# 推广中心真实业务联调清单

> 目标：验证“邀请 -> 绑定 -> 激活 -> 日佣金 -> 日结到真实余额 -> 撤销/调整 -> 权限安全”整条链路。

## A. 基础配置联调

1. 进入 `/admin/promotion`
2. 在“等级与返利配置”中配置至少 4 个等级：
   - Lv1：0 人
   - Lv2：5 人
   - Lv3：15 人
   - Lv4：30 人
3. 配置一级/二级返利比例
4. 配置：
   - 激活阈值：5 USD
   - 激活奖励：例如 1 USD
   - 每日结款时间：00:00
5. 配置 4 段规则模板
6. 打开用户 `/promotion`，确认规则模板已实时渲染

## B. 邀请绑定联调

1. 用户 A 打开 `/promotion`
2. 记录唯一推广码与推广链接
3. 使用新的浏览器会话访问 `.../register?ref=<A的邀请码>`
4. 用户 B 注册成功
5. 在 `/admin/promotion` 确认：
   - B 已绑定到 A
   - A 的下级数量 +1
6. 验证异常情况：
   - 用户不能绑定自己
   - 已绑定用户不能再次自助改绑
   - 用户一旦产生消费，不能再自助绑定

## C. 激活联调

1. 让 B 产生真实消费：
   - 累计 `real_actual_cost = 5.00` 时，确认 **不激活**
   - 累计 `real_actual_cost = 5.01` 时，确认 **激活**
2. 检查：
   - `promotion_activations` 写入成功
   - A 获得一条 `activation` 类型的待结算记录
   - A 的“直推已激活人数”变化
   - 若等级门槛达成，A 在用户推广中心的 Level Status Card 显示新等级

## D. 日佣金联调

1. 在同一业务日内，让 B 产生多笔真实消费
2. 如果存在 A -> B -> C 的链路，再让 C 产生消费
3. 检查：
   - 当天用户 `/promotion` 的收益不是即时到账
   - 后台 `/admin/promotion` 佣金列表出现待结算记录
   - 一级返佣、二级返佣金额符合等级比例
4. 到次日结款时间后，检查：
   - 佣金状态从 `pending` 变为 `settled`
   - 收益进入 A 的**真实余额**
   - 余额历史中新增 `promotion_balance` 正向流水

## E. 撤销 / 调整联调

### 待结算佣金撤销
1. 选择一条 `pending` 佣金记录
2. 点击撤销
3. 验证：
   - 记录状态变为 `cancelled`
   - 用户真实余额 **不变**

### 已结算佣金撤销
1. 选择一条 `settled` 佣金记录
2. 点击撤销
3. 验证：
   - 用户真实余额被回扣
   - 余额历史写入 `promotion_balance` 负向流水
   - 佣金状态变为 `cancelled`

### 手工奖励 / 差额调整
1. 在“佣金管理”中发放 `manual` / `promotion` / `adjustment`
2. 验证：
   - 记录立即为 `settled`
   - 用户真实余额同步变化

## F. 前端交互联调

### 用户侧
1. `/promotion` 4 个 tab 可切换
2. 邀请推广：
   - 推广码唯一
   - 可复制推广码/链接
   - 海报可下载 PNG（失败时回退 SVG）
3. 我的团队：
   - 默认每页 10 条
   - 默认今日贡献降序
   - 可切换分页大小、排序字段、排序方向
   - 设置保存在浏览器，下次打开仍保持
4. 收益明细：
   - 动作列显示“发奖励 / 撤销奖励”
   - 状态列显示“待结算 / 已结算 / 已撤销”

### 管理侧
1. 关系管理：
   - 点击查看可加载右侧链路
   - 右侧顺序为：`当前查看 -> 一级上级 -> 二级上级`
   - 节点显示邮箱、等级、总提成比例、当前链路返利比例
   - 右侧直接下级列表可执行“移除”
2. 等级配置：
   - 新增等级后，用户达标即显示
3. 规则模板：
   - 编辑后用户侧规则区实时变化

## G. 安全联调

1. 未登录访问 `/promotion`，应跳转登录
2. 普通用户访问 `/admin/promotion`，应被拒绝
3. 普通用户直接请求：
   - `GET /api/v1/admin/promotion/dashboard`
   - `POST /api/v1/admin/promotion/commissions/manual-grant`
   - `DELETE /api/v1/admin/promotion/relations/:id/downlines/:id`
   均应返回 401/403
4. 验证普通用户无法越权：
   - 改别人推广关系
   - 发放佣金
   - 修改等级与返利配置

## 验收标准

- 推广关系、佣金记录、真实余额三方一致
- 用户侧与管理侧显示一致
- 结算为按日聚合、次日统一发放
- 不存在未授权访问或越权修改漏洞
